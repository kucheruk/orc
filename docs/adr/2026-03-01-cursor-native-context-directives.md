# ADR: Cursor-native контекст вместо auto-inject файлов

## Статус
Принято (2026-03-01)

## Контекст
- Ранее рассматривался подход auto-inject: инлайнить полное содержимое `AGENTS.md` и `techspec.md` в каждый prompt.
- Такой подход увеличивает входной prompt и может создавать значимый token burn на каждом старте/рестарте задачи.
- Для Cursor CLI и Agent уже доступны нативные механизмы контекста: `AGENTS.md`, project rules, чтение файлов по `@...`, semantic retrieval.

## Решение
- Отказываемся от стратегии auto-inject полного содержимого проектных документов в prompt.
- В prompt-шаблонах оставляем только краткие директивы для чтения `@AGENTS.md` и `@techspec.md`.
- Runtime helper для inline-вставки файлов удаляется как неиспользуемый и рискованный для повторного включения.
- Базовый принцип: сначала Cursor-native retrieval, потом точечные чтения по необходимости.

## Альтернативы
### A1. Оставить auto-inject (отклонено)
- Плюсы: агент сразу видит весь контекст.
- Минусы: постоянные лишние входные токены и ухудшение масштабируемости при росте `techspec.md`.

### A2. Частичный inject (отклонено)
- Плюсы: меньше токенов, чем при полном inject.
- Минусы: появляется сложная логика отбора фрагментов и риск рассинхронизации с источниками правды.

### A3. Cursor-native директивы (выбрано)
- Плюсы: минимальный prompt, лучшее использование нативного кэшируемого контекста IDE, проще сопровождение.
- Минусы: агенту нужно сделать явный шаг чтения, если задача реально зависит от детали из документа.

## Последствия
### Положительные
- Снижается стабильная входная token-нагрузка на стартах и рестартах.
- Prompt становится короче и проще для сопровождения.
- Убирается мёртвый код, который мог быть случайно активирован в будущем.

### Компромиссы
- При задачах с жёсткой зависимостью от деталей из `techspec.md` агенту нужно явно прочитать соответствующий файл.

## Инварианты
- `AGENTS.md` и `techspec.md` остаются источниками правды, но не инлайн-частью каждого prompt.
- Приоритет остаётся за репозиторными правилами и реальным кодом.
