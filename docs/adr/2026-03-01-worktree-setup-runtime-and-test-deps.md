# ADR: Единый setup worktree для runtime и тестов

## Контекст

В проекте отсутствовал единый setup-скрипт для Cursor worktree, а часть обязательных зависимостей
не была описана в `pyproject.toml` (`psutil` для runtime, `pytest` для тестов).

Это приводило к нестабильному старту в новых worktree:

1. `pytest` мог отсутствовать, и проверка изменений ломалась на окружении.
2. Runtime мог падать на `ModuleNotFoundError: psutil`.
3. Команда настройки отличалась от сессии к сессии.

## Решение

Добавлен идемпотентный скрипт `scripts/worktree-setup.sh`:

- выполняет `uv sync --dev`;
- валидирует, что доступны обязательные пакеты: `rich`, `textual`, `psutil`, `pytest`;
- завершаетcя с ошибкой при неполной установке.

Также зависимости зафиксированы в `pyproject.toml`:

- `psutil` — в runtime dependencies;
- `pytest` — в `dependency-groups.dev`.

## Альтернативы

1. Оставить ручной `uv sync` без валидации.
   - Плюсы: меньше кода.
   - Минусы: нет гарантии полного окружения.

2. Устанавливать всё через ad-hoc `uv pip install ...`.
   - Плюсы: быстро локально.
   - Минусы: дрейф окружений, отсутствие декларативности.

3. Единый setup-скрипт + декларативные зависимости (выбрано).
   - Плюсы: повторяемость, предсказуемость, быстрый health-check окружения.
   - Минусы: нужно поддерживать список обязательных модулей.

## Последствия

- Подготовка worktree стала воспроизводимой и проверяемой.
- Снизился риск ложных падений из-за окружения.
- Ускорился старт новых сессий и проверка тестов.
