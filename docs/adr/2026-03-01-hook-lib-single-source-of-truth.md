# ADR: единый источник правды для hook runtime

## Статус
Принято (2026-03-01)

## Контекст
- В `orc_core/hooks.py` код hook runtime (`orc_hook_lib`, `orc_before_submit`, `orc_stop`) был продублирован как большие inline-строки.
- Параллельно существовали файловые версии в `orc_core/hook_scripts/`, что создавало два источника правды.
- Любая доработка в одном месте легко расходилась с другим и повышала риск регрессий.

## Решение
- Канонический источник hook runtime — файловые скрипты в `orc_core/hook_scripts/`.
- `ensure_repo_hooks()` в `orc_core/hooks.py` больше не содержит inline-двойников и рендерит скрипты только из файлов:
  - `orc_core/hook_scripts/orc_before_submit.py`
  - `orc_core/hook_scripts/orc_stop.py`
  - `orc_core/hook_scripts/orc_hook_lib.py`
- Placeholder `__ORC_ROOT__` подставляется при рендере.
- Запись в `<workspace>/.cursor/hooks/*.py` делается идемпотентно: файл перезаписывается только если контент изменился.

## Последствия
### Положительные
- Убран архитектурный долг «inline + файл» и снижен риск рассинхронизации логики.
- Обновления hook runtime выполняются в одном месте и легче ревьюятся.
- Поведение генерации стало более предсказуемым и идемпотентным.

### Компромиссы
- Появляется жёсткий контракт на структуру `orc_core/hook_scripts/`; удаление/переименование файлов ломает генерацию.

## Инварианты
- `ensure_repo_hooks_config()` продолжает регистрировать команды вида `python3 <hook_path>` в `.cursor/hooks.json`.
- Hook-скрипты исполняются из `<workspace>/.cursor/hooks/`.
- Fallback на inline-скрипты отсутствует: единственный источник правды — файловые скрипты.
