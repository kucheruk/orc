# ADR: Автоинжект контекста AGENTS и techspec в prompt

## Контекст

Headless-запуск через Cursor CLI не гарантирует, что агент сразу видит полный контекст проектных правил и ограничений.
В результате агент тратит первые шаги на чтение `AGENTS.md` и `techspec.md` через tool calls, что увеличивает latency и расход токенов на каждой задаче.

Нужно обеспечить zero-shot контекст: правила и ограничения должны попадать в prompt автоматически до старта выполнения.

## Решение

Принято решение централизованно инжектить содержимое проектных файлов в prompt на стороне ORC runtime:

- перед запуском `agent -p` в prompt добавляется блок `<project_context ...>` с содержимым `AGENTS.md` и `techspec.md`;
- инжект выполняется единым helper для всех путей: initial/start, continue/restart и commit phase;
- добавлены CLI-флаги управления:
  - `--inject-agents-md` / `--no-inject-agents-md`
  - `--inject-techspec-md` / `--no-inject-techspec-md`
  - `--agents-path`, `--techspec-path`
- политика по умолчанию: `default-on-by-presence` (включать инжект только если файл найден);
- если файл отсутствует, ORC выводит оранжевое предупреждение и продолжает работу без падения процесса.

## Альтернативы

1. Оставить текущую модель “агент сам читает файлы”.
   - Плюсы: не менять runtime.
   - Минусы: постоянный token burn, медленный холодный старт задачи.

2. Инжектить только в `prompts/default.txt`.
   - Плюсы: проще реализация.
   - Минусы: не покрывает continue/restart и commit phase; поведение непоследовательно.

3. Инжект в runtime (выбран).
   - Плюсы: единое поведение во всех фазах, меньше лишних tool calls, быстрый старт.
   - Минусы: prompt становится длиннее; требуется аккуратная диагностика missing-файлов.

## Последствия

- Агент получает “правила игры” до первого шага и реже делает служебные чтения документов.
- Улучшается предсказуемость поведения между initial/restart/commit.
- Появляется явная диагностируемость отсутствующих файлов контекста (оранжевое предупреждение + WARN лог).
- Для нестандартных репозиториев сохраняется гибкость через path/enable CLI-флаги.
