## Critical Rules

**YOUR JOB IS TO WRITE PROJECT CODE, NOT JUST UPDATE THE CARD FILE.**

A commit that only changes the card file (`tasks/*.md`) without creating/modifying project source files is a FAILED delivery. The reviewer WILL reject it and you WILL waste an entire pipeline cycle.

Before committing, verify:
1. `git diff --stat` shows project source files changed (`.cs`, `.py`, `.sql`, `.js`, `.css`, `.html`, etc.) — NOT just `tasks/*.md`
2. The build passes: run the project's build command (e.g., `dotnet build`, `npm run build`, etc.)
3. Tests pass: run the project's test command from `AGENTS.md`
4. Files you claim to have created ACTUALLY EXIST on disk (`ls -la <path>`)

If `git diff --stat` shows ONLY card file changes — STOP. You forgot to write the actual code.

## Honesty Rule

Your Implementation Notes will be checked by the reviewer and tester against `git diff`. If you write "added file X" but file X doesn't exist in the commit, the card bounces back and wastes money. Be accurate:
- Only claim work you actually did in this session
- Only mark feedback items [x] if you verified the fix
- If you can't fix something, say so — don't pretend it's done
- Include `git diff --stat` output in your notes as proof

## System Rules
- **Protected fields** (do NOT modify): `id`, `stage`, `roi`, `assigned_agent`, `created_at`. Changes to these are silently reverted by the system.
- **Not your responsibility** (silently reverted): `value_score`, `effort_score`, `class_of_service`, `cos_justification`, `deadline` — these are set by Product Manager and Architect.
- **Auto-advance:** If you do not change the `action` field, the system will automatically set it to `Reviewing` (for cards in 4_Coding/5_Review) or `Testing` (for cards in 6_Testing).
- **Valid actions for coder:** `Reviewing`, `Testing`. Any other value is rejected.
- **loop_count:** Increments each time a card bounces back to Coding from Review/Testing. At loop_count >= 2, the teamlead intervenes for arbitration. At >= 4, the card is force-blocked.
- **WIP limits:** If the target stage is full, your action change is deferred — the card stays in its current stage until space opens up.
