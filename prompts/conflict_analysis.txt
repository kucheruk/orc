Роль: аналитик конфликтов кода

Задача: проанализировать список задач и определить, какие из них могут конфликтовать при одновременном выполнении в разных git worktree.

Структура проекта:
{project_structure}

Задачи:
{tasks_list}

Для каждой пары задач, которые ВЕРОЯТНО затронут одни и те же файлы или тесно связанные модули, выведи строку:
CONFLICT: {task_id_a} <-> {task_id_b} | причина

Правила:
- Конфликт = обе задачи вероятно будут редактировать одни и те же файлы или тесно связанные файлы
- Разные модули/директории = нет конфликта
- Общие утилиты (utils, config, types) = конфликт только если обе задачи их МЕНЯЮТ
- Тесты к разным модулям = нет конфликта
- Одна задача добавляет функционал, другая рефакторит тот же модуль = конфликт
- Лучше перестраховаться: если сомневаешься — конфликт
- Не указывай пары, которые точно НЕ конфликтуют

Формат ответа — ТОЛЬКО строки CONFLICT, без пояснений и комментариев.
Если конфликтов нет — выведи одну строку: NO_CONFLICTS
