Роль: эксперт-программист

Задача: подготовить и сделать git commit по результатам выполненной задачи.

Контекст:
- Только что была завершена задача из @{backlog}
- Task ID: {task_id}
- Task text: {task_text}
- Workspace: {workspace}
- SDLC артефакты:
  - planning: `{artifact_plan}`
  - design: `{artifact_design}`
  - implementation: `{artifact_implementation}`
  - review: `{artifact_review}`
  - testing: `{artifact_testing}`

Ограничения:
- В этой фазе НЕ меняй код, тесты, backlog или документацию (кроме случаев, когда commit невозможно сделать без минимальной правки формата сообщения/метаданных).
- Твоя цель: корректный commit. Никакой “доделки задачи”.

Алгоритм:
1) Быстро собери контекст репозитория:
   - `git status`
   - `git diff`
   - `git diff --staged`
   - `git log -10 --oneline`

2) Если нет изменений для коммита (working tree чистый и нет staged):
   - Напиши коротко: “skip commit: clean tree”.
   - Заверши работу.

3) Если есть изменения:
   - Стадируй всё: `git add -A`
   - Сформируй commit message в строгом формате Conventional Commits.
   - Формат: `<type>[optional scope]: <description>`
   - Допустимые типы (type): feat, fix, docs, style, refactor, perf, test, build, ci, chore, revert.
   - ОБЯЗАТЕЛЬНО укажи `{task_id}`. Его можно добавить в scope (например, `feat({task_id}): ...`) или в footer (например, `Refs: {task_id}`).

4) Сделай commit (сообщение через heredoc, чтобы не сломать кавычки), например:
   git commit -m "$(cat <<'EOF'
   feat: <краткое описание в повелительном наклонении>

   <Опциональное более подробное описание того, что было сделано и почему>

   Refs: {task_id}
   EOF
   )"

5) После коммита:
   - `git status` должен быть чистым (или объясни, что осталось и почему).
   - Не делай `git push`, если явно не просили.

Вывод в конце:
- 1-2 строки: что закоммитил (смысл) + hash коммита (краткий) или “skip commit”.

## SDLC artifact output (обязательно)
Перед завершением этой фазы:
1) Запиши итог handoff в `{artifact_handoff}` (не пустой файл), опираясь на все входные артефакты.
2) Обнови/создай `tasks/{task_id}.md` как человекочитаемый итог задачи.
