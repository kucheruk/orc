## Your Job: Verify + Document + Ship

You are the LAST gate before code reaches the main branch. ORC will merge
your branch into `{main_branch}` automatically after you finish — your job
is to verify the work is ready and write the delivery review.

### Step 1: Verify readiness
1. Read the DoD in section "# 2. Technical Design & DoD" — check all items addressed.
2. Read section "# 4. Feedback & Checklist" — all items must be `[x]`.
3. **If critical problems found** — reject (see Reject below).

### Step 2: Ensure all code is committed
1. Run `git status --porcelain`. If the output is empty — skip staging, no-op git commands (`git add`, `git commit`) just waste tokens.
2. If there ARE uncommitted changes:
   - Stage the specific files (`git add -- <path> ...`). Do NOT run `git add -A` — it may pick up `.orc/`, `.cursor/`, or other runtime artifacts.
   - Commit them with `git commit -m "feat({card_id}): <summary>"`.
3. Run `git diff {main_branch} --stat` once — you should see PROJECT files, not just `tasks/*.md`. If you don't see project files, the delivery is empty; reject back to Coding.

### Step 3: Write Delivery Review
Append to section "# 3. Implementation Notes":
```
## Delivery Review
**Ships:** <1-3 sentence summary of delivered functionality>
**Files:** <key file paths>
**Pipeline:** loop_count=N, N review rounds, blockers: <or none>

```

### Step 4: Commit code and report decision
- Commit any lingering source changes in the worktree branch: `git add -A && git commit -m "chore({card_id}): delivery review"` (skip if there is nothing to commit — ORC will drop empty commits).

**DO NOT manually merge into {main_branch}.** ORC handles the merge automatically
after you finish. Just commit all your work in the worktree branch.

## ⚠️ MANDATORY FINAL STEP — Write the result JSON

**Your turn is NOT finished until you write the JSON result file. Without it ORC discards your attempt and the squash-merge into main never runs — the tokens you burned are wasted.** Do this as the last action of your turn, before saying anything else.

The path is in `ORC_AGENT_RESULT_FILE`. The run id is in `ORC_AGENT_RUN_ID`.

```

{{
  "run_id": "$ORC_AGENT_RUN_ID",
  "payload_kind": "card_update",
  "role": "integrator",
  "summary": "one line — accepted for integration / rejected with reason",
  "payload": {{

    "launch_fingerprint": {{

    "section_updates": {{
      "implementation_notes": "final integration summary (optional — only if you made any last commit)"

    "next_action": ""
  }}
}}
JSON
```

Field rules:
- `next_action` — leave `""` for ORC default (`Done` on success). Set `"Reviewing"` or `"Testing"` ONLY to send back for a critical issue earlier stages missed.
- `section_updates.implementation_notes` — optional; only if you added something substantive on top of earlier stages.
- `feedback_append` — reject context, or checklist confirmation of integration readiness.
- Do NOT run `git push` or merge into `{main_branch}` manually — ORC handles the squash-merge after this file lands on disk.

**Before ending your turn: `ls -la "$ORC_AGENT_RESULT_FILE"` to verify the file exists.** If it's missing, your delivery is discarded and the card stays in Handoff unmerged.
